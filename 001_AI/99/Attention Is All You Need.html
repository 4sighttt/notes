      • Pattern Weaver 관점으로 정리한 개인 지식 체계 운영안
      • 인간은 단순한 자극-반응 기계가 아니라 정보 처리 시스템이다
      • 지능은 어떻게 정보를 처리하는가
      • 휴리스틱(Heuristic)

    • Transformer 구조 핵심

    • Self-Attention 작동 원리

    • 왜 이 논문이 AI 역사에서 패러다임 전환인지

    요약

    • 이 논문의 핵심은 “순서를 따라가며 읽는 모델(RNN)”을 버리고, 모든 토큰이 서로를 직접 참고하게 하자는 것이다.

    • 그 구현이 Self-Attention이고, 전체 구조가 Transformer다.

    • 이 구조는 병렬화가 잘 되고, 긴 거리 의존성도 더 짧은 경로로 다룬다.

    • 논문은 번역 성능에서도 당시 최고 수준을 기록했다.

    • 그래서 이 논문은 단순 성능 향상이 아니라, 딥러닝의 주력 설계 원리 자체를 바꾼 논문으로 평가된다.

    1. 먼저 큰 그림

    이 논문 이전의 자연어 처리 주류는 대체로 이런 식이었다.

    • 문장을 왼쪽에서 오른쪽으로 한 칸씩 읽는다

    • 이전 상태를 다음 상태로 넘긴다

    • 즉, 계산이 순차적이다

    이 방식의 대표가 RNN, LSTM, GRU 계열이다. 논문도 당시 state of the art가 이런 recurrent 계열이었다고 설명한다. 그런데 이런 구조는 계산이 시간축에 묶여 있기 때문에, 긴 문장일수록 병렬화가 어렵고 멀리 떨어진 단어 관계를 학습하는 데 구조적 불리함이 있다. 저자들은 바로 이 지점을 문제로 본다.

    논문의 한 문장으로 압축하면 이렇다.

    순차 처리 없이 attention만으로도 충분하지 않은가?

    그리고 그 답으로 나온 것이 Transformer다. 저자들은 Transformer를 recurrence와 convolution을 entirely 버리고 attention만으로 구성한 구조라고 제시한다.

    2. Transformer 구조 핵심

    Transformer는 겉으로 보면 인코더-디코더 구조다.
    즉,

    • Encoder: 입력 문장을 표현으로 바꾼다

    • Decoder: 그 표현을 보고 출력 문장을 한 토큰씩 만든다

    이 전체 골격은 기존 seq2seq 계열과 비슷하다. 바뀐 것은 안에서 작동하는 엔진이다. 기존에는 RNN이나 CNN이 핵심 엔진이었는데, 여기서는 그 자리를 self-attention + feed-forward가 차지한다.

    2-1. Encoder

    논문 기본 모델의 encoder는 동일한 층 6개를 쌓는다. 각 층은 두 부분으로 이루어진다.

    1. Multi-Head Self-Attention

    2. Position-wise Feed-Forward Network

    그리고 각 부분 바깥에는

    • residual connection

    • layer normalization

    이 붙는다. 즉 흐름은 대략 이렇다.

    입력 → self-attention → 잔차연결+정규화 → FFN → 잔차연결+정규화

    이렇게 6층 반복이다.

    2-2. Decoder

    decoder도 역시 6개 층을 쌓는데, encoder보다 한 단계 더 있다.

    1. Masked Multi-Head Self-Attention

    2. Encoder-Decoder Attention

    3. Feed-Forward Network

    왜 첫 attention이 masked냐면, 출력 문장을 생성할 때 미래 단어를 미리 보면 안 되기 때문이다.
    예를 들어 5번째 단어를 예측할 때 6번째 단어를 이미 참고하면 cheating이 된다. 그래서 decoder self-attention은 현재 위치보다 뒤쪽 토큰을 가리는 mask를 둔다. 논문도 이 masking이 autoregressive 성질을 보장한다고 설명한다.

    2-3. Positional Encoding

    Transformer는 recurrence도 convolution도 없다.
    그러면 모델 입장에서는

    • “나는”

    • “학교에”

    • “간다”

    가 어떤 순서인지 알 방법이 없다.

    그래서 논문은 입력 임베딩에 positional encoding을 더한다.
    기본 버전은 학습형이 아니라 사인/코사인 함수 기반의 고정 위치 인코딩이다. 저자들은 이것이 상대적 위치를 다루는 데 유리할 수 있다고 봤고, 실험상 learned positional embedding과 성능은 거의 비슷했다고 보고한다.

    핵심은 이것이다.

    Transformer는 순서를 “계산의 순차성”으로 표현하지 않고,
    벡터에 추가된 위치 정보로 표현한다.

    이게 매우 큰 발상 전환이다.

    3. Self-Attention 작동 원리

    이 논문의 진짜 중심은 여기다.

    3-1. 직관

    문장 속 각 단어가 문장의 다른 모든 단어를 보고,
    “지금 내 의미를 만들 때 누구를 얼마나 참고해야 하지?”를 계산한다.

    예를 들어 문장

    “The animal didn’t cross the street because it was too tired.”

    여기서 it의 의미를 파악하려면 animal을 강하게 참고해야 한다.
    Self-attention은 이런 참조 강도를 직접 계산한다.

    즉, 각 토큰은 다른 토큰들을 향해 가중치를 만들고, 그 가중 평균으로 새 표현을 만든다.

    3-2. Q, K, V

    논문은 attention을 Query, Key, Value로 설명한다.

    각 토큰 벡터에서 선형변환을 통해 세 가지를 만든다.

    • ==Q (Query): 내가 지금 무엇을 찾고 있는가==

    • ==K (Key): 나는 어떤 특징을 가지고 있는가==

    • ==V (Value): 내가 실제로 전달할 정보는 무엇인가==

    어떤 토큰 A가 토큰 B를 얼마나 참고할지는
    A의 Query와 B의 Key가 얼마나 잘 맞는지로 계산한다.

    즉,

    • Q와 K의 내적(dot product)을 계산

    • 값이 클수록 관련성 높음

    • softmax로 정규화하여 가중치로 바꿈

    • 그 가중치로 V들을 가중합

    논문의 식은 다음과 같다.

    [
    Attention(Q,K,V)=softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
    ]

    즉 핵심은
    “관련성 점수 → 확률화 → 정보 혼합” 이다.

    3-3. 왜 (\(\sqrt{d_k}\)) 로 나누는가

    내적 차원이 커지면 값이 너무 커질 수 있다.
    그러면 softmax가 지나치게 뾰족해지고 gradient가 작아져 학습이 불안정해진다.

    그래서 저자들은 dot product를 (\(\sqrt{d_k}\))로 나눈다.
    이것이 Scaled Dot-Product Attention이다.

    이건 단순한 수학적 장식이 아니라 학습 안정성 장치다.

    3-4. Self-Attention이 “self”인 이유

    Q, K, V가 모두 같은 시퀀스에서 나온다.
    즉 문장 내부 토큰들이 서로를 참조한다.

    • encoder self-attention: 입력 문장 내부에서 서로 참조

    • decoder self-attention: 출력 문장 내부에서 과거 토큰들만 참조

    • encoder-decoder attention: 출력 쪽 토큰이 입력 쪽 토큰을 참조

    그래서 attention이 세 군데 쓰이지만,
    그중 핵심은 같은 시퀀스 안에서 관계를 계산하는 self-attention이다.

    4. Multi-Head Attention은 왜 필요한가

    한 번의 attention만 하면 모델은 하나의 기준으로만 관계를 본다.

    그런데 실제 언어 관계는 여러 종류가 있다.

    • 문법적 관계

    • 의미적 관계

    • 지시어 관계

    • 장거리 의존성

    • 구문 경계

    • 강조 대상

    그래서 논문은 attention을 한 번 하지 않고, 여러 개 head로 병렬 수행한다.
    각 head는 서로 다른 선형변환을 통해 다른 표현 공간에서 관계를 본다.
    이 결과들을 concat한 뒤 다시 선형변환한다.

    직관적으로 말하면:

    • 1번 head는 주어-동사 관계를 볼 수 있고

    • 2번 head는 수식 관계를 볼 수 있고

    • 3번 head는 멀리 떨어진 단어 연결을 볼 수 있다

    논문도 appendix의 attention visualization에서, 서로 다른 head가 다른 역할을 학습한 것처럼 보인다고 보여준다. 예를 들어 장거리 의존성이나 대명사 해소와 비슷한 패턴이 나타난다.

    중요한 점은, 이 논문이 attention을 단순 보조장치가 아니라 표현 학습의 주 엔진으로 승격시켰다는 것이다.

    5. Feed-Forward Network는 왜 필요한가

    attention만 있으면 토큰 간 정보 섞기는 잘 되지만,
    각 위치에서 비선형 변환을 통해 표현을 더 정교하게 바꾸는 단계도 필요하다.

    그래서 각 층마다 attention 뒤에 position-wise FFN이 붙는다.
    논문 식은

    [
    FFN(x)=max(0, xW_1+b_1)W_2+b_2
    ]

    이다.
    즉 각 위치마다 독립적으로 같은 MLP를 적용한다.

    대략 이렇게 보면 된다.

    • attention: 토큰들끼리 정보를 교환

    • FFN: 각 토큰 표현을 내부적으로 가공

    이 둘이 번갈아 쌓이면서 모델이 깊어진다.

    6. 왜 Self-Attention이 RNN보다 유리했는가

    논문은 self-attention의 장점을 세 기준으로 비교한다.

    6-1. 병렬화

    RNN은 본질적으로 순차적이다.

    • 1번째 hidden state를 계산해야

    • 2번째를 계산할 수 있고

    • 2번째를 계산해야 3번째를 계산할 수 있다

    반면 self-attention은 한 층 안에서 모든 위치 쌍의 관계를 한 번에 계산할 수 있다.
    즉 GPU 친화적이다.

    논문은 self-attention layer의 sequential operation이 O(1), recurrent layer는 O(n)이라고 비교한다.

    이건 엄청난 차이다.
    실제로 대규모 학습 시대에는 “계산 그래프가 얼마나 병렬화되느냐”가 성능 못지않게 중요하다.

    6-2. 장거리 의존성

    RNN에서 멀리 떨어진 두 단어가 상호작용하려면, 중간 많은 단계를 거쳐야 한다.

    예를 들어 문장 맨 앞 단어와 맨 뒤 단어의 관계는
    정보가 긴 경로를 지나야 하므로 희석되기 쉽다.

    반면 self-attention에서는 모든 토큰이 서로 직접 연결된다.
    그래서 최대 path length가 O(1)이다.
    RNN은 O(n), convolution은 보통 O(log n) 수준이다.

    이건 언어처럼 멀리 떨어진 요소가 갑자기 연결되는 문제에서 특히 중요하다.

    6-3. 계산 효율

    self-attention은 (O(n^2 d))라서 시퀀스 길이가 아주 길면 비용이 커진다.
    하지만 논문은 당시 기계번역에서 흔한 문장 길이에서는, 표현 차원 (d)보다 길이 (n)이 작은 경우가 많아 recurrent보다 유리하다고 논한다.

    즉 이 논문은 “무조건 모든 경우에 attention이 최고”라고 말하지 않는다.
    오히려 꽤 정직하게 trade-off도 적어둔다.

    이 점이 중요하다.
    당시에는 attention이 마법이라서가 아니라, 실제 계산 구조와 하드웨어 환경에 맞는 설계였던 것이다.

    7. 실험 결과가 왜 강력했는가

    좋은 아이디어 논문과 패러다임 전환 논문의 차이는 보통 여기에 있다.
    아이디어만 새롭다고 판이 바뀌지는 않는다.
    성능 + 효율 + 일반화 가능성이 함께 나와야 한다.

    이 논문은 그걸 보여줬다.

    7-1. 번역 성능

    논문 초록과 결과표에 따르면,

    • WMT 2014 English-to-German에서 28.4 BLEU

    • 기존 최고 기록 대비 2 BLEU 이상 향상

    • WMT 2014 English-to-French에서 41.8 BLEU

    • 그것도 기존 최고 모델 대비 훨씬 적은 훈련 비용

    을 기록했다.

    즉 메시지는 분명했다.

    attention-only 구조가
    단지 “될 수도 있다”가 아니라
    실제로 더 잘 된다.

    7-2. 학습 시간

    저자들은 base model이 8개의 P100 GPU에서 12시간, big model이 3.5일 훈련되었다고 적는다.

    여기서 중요한 건 절대 시간이 아니라 상대 비교다.
    기존 강력한 번역 모델들에 비해 훨씬 적은 비용으로 최고 수준 결과를 냈다는 점이 중요하다.
    즉 연구자와 산업계 모두에게 매력적인 구조였다.

    7-3. 다른 과제로의 일반화

    논문은 기계번역만 하지 않고, English constituency parsing에도 Transformer를 적용했다.
    완전한 task-specific tuning 없이도 강한 결과를 냈다고 보고한다.

    이건 매우 중요하다.

    왜냐하면 패러다임 전환은 대개
    “한 문제를 잘 푸는 기술”이 아니라
    “여러 문제에 이식 가능한 일반 원리”일 때 일어나기 때문이다.

    Transformer는 바로 그 일반성을 보여줬다.

    8. 왜 이 논문이 패러다임 전환이었는가

    이 부분이 가장 중요하다.

    8-1. 보조 메커니즘이 주인공이 됨

    이 논문 이전에도 attention은 있었다.
    하지만 대개는 RNN 위에 붙는 보조장치였다.

    즉 기존 관점은:

    • 본체는 RNN

    • attention은 도우미

    이 논문은 그 위계를 뒤집었다.

    • 본체는 attention

    • recurrence는 필요 없음

    이건 기술적 개선이 아니라 설계 철학의 전복이다.

    8-2. “순서를 따라 계산해야 한다”는 고정관념 붕괴

    언어는 순서가 중요하니, 모델도 순차적으로 읽어야 한다는 생각이 강했다.

    Transformer는 말한다.

    • 순서를 계산 순서로 표현할 필요는 없다

    • 위치 정보는 따로 주입하면 된다

    • 의미 계산은 전역 상호참조로 해도 된다

    즉,

    언어의 순서성 ≠ 모델의 순차 계산

    이 분리가 엄청나게 중요했다.

    이 순간부터 연구자들은
    “언어를 다루는 모델은 반드시 시간축을 따라야 한다”는 직관에서 벗어나게 된다.

    8-3. 규모 확장에 유리한 구조

    패러다임 전환은 “더 크게 만들기 쉬운가”와 강하게 연결된다.

    Transformer는 병렬화가 잘 되고, GPU/TPU 자원을 더 잘 활용할 수 있다.
    이것은 나중에 모델 규모를 폭증시키는 길을 열었다.
    즉 이 논문은 단지 번역 논문이 아니라, 거대 모델 시대의 계산적 토대를 만든 셈이다.

    여기서 중요한 포인트는 다음이다.

    • RNN은 이론적으로 가능해도 대규모 확장이 상대적으로 불리하다

    • Transformer는 구조적으로 행렬연산 중심이라 하드웨어와 잘 맞는다

    • 그래서 “좋은 모델”을 넘어서 “확장 가능한 모델”이 되었다

    5. 이 구조가 병렬화와 확장성에서 유리했기 때문에 패러다임이 바뀌었다.

    요약

    1. Multi-Head Attention은 한 가지 관계만 보는 것을 막기 위해 여러 관점에서 동시에 attention을 계산한다.

    2. Self-Attention은 모든 토큰이 서로 직접 연결되므로 장거리 의존성을 더 짧은 경로로 학습한다.

    3. RNN은 계산이 순차적이지만 Self-Attention은 병렬 계산이 가능하다.

    4. 그래서 대규모 학습과 모델 확장에서 Transformer가 유리했다.

    5. GPT는 Transformer의 Decoder 부분만 사용하는 구조다.

    6. 이유는 GPT의 목표가 다음 토큰 예측(next token prediction)이기 때문이다.

    1. 왜 Multi-Head Attention이 필요한가

    Self-Attention을 한 번만 사용하면 문제가 생긴다.

    모델이 단 하나의 기준으로만 단어 관계를 보게 된다.

    예문

    The animal didn’t cross the street because it was tired

    여기서 it은 animal을 가리킨다.

    하지만 문장을 이해하려면 여러 종류의 관계가 있다.

    예를 들어

    • 문법 관계 (주어-동사)

    • 의미 관계

    • 대명사 참조

    • 수식 관계

    • 장거리 의존성

    한 번의 attention은 이런 관계 중 한 종류만 잘 학습할 가능성이 높다.

    그래서 논문은 attention을 여러 개로 나눈다.

    head1 → 문법 관계
head2 → 의미 관계
head3 → 대명사 관계
head4 → 장거리 의존성
...

    이렇게 여러 attention을 병렬로 계산한다.

    논문에서는 보통 8개 head를 사용한다.

    그리고 마지막에

    Concat(head1, head2, ... head8)
→ 선형변환

    으로 합친다.

    핵심 아이디어

    하나의 attention = 하나의 관점
Multi-Head = 여러 관점 동시 학습

    2. 왜 Self-Attention이 RNN보다 근본적으로 강한가

    논문이 제시한 핵심 이유는 3가지다.

    1️⃣ 장거리 의존성

    문장

    The book that the professor who the student met recommended is interesting

    book 과 interesting은 멀리 떨어져 있다.

    RNN

    정보 전달 방식

    book → → → → → → → → interesting

    경로 길이

    O(n)

    그래서 정보가 점점 희석된다.

    Self-Attention

    모든 토큰이 직접 연결된다.

    book ↔ interesting

    경로 길이

    O(1)

    그래서 멀리 떨어진 단어 관계를 더 쉽게 학습한다.

    2️⃣ 병렬 계산

    RNN

    h1 → h2 → h3 → h4

    반드시 순서대로 계산해야 한다.

    그래서 GPU를 충분히 활용하기 어렵다.

    Self-Attention

    모든 관계를 한 번의 행렬연산으로 계산한다.

    QKᵀ

    그래서

    모든 토큰 관계를 동시에 계산

    가능하다.

    논문에서도

    self-attention sequential operations = O(1)
recurrent = O(n)

    이라고 비교한다.

    3️⃣ 표현 방식의 차이

    RNN

    문장의 의미 = hidden state

    즉 과거 요약 상태

    Self-Attention

    문장의 의미 = 토큰 간 관계 구조

    즉 관계 그래프

    이 구조가 더 유연하다.
