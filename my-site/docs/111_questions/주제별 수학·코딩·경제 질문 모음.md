# 주제별 수학·코딩·경제 질문 모음

사유와 학습의 출발점이 되는 질문들을 대분류(3)·중분류(30)·세부 질문 단위로 정리한 문서.

---

## 1. 수학

### 1-1. 수와 연산

수는 세계에 실제로 존재하는 것인가, 아니면 인간이 세계를 세기 위해 만든 추상적 도구인가?  
자연수, 정수, 유리수, 실수, 복소수는 각각 어떤 필요에 의해 등장했는가?  
음수와 허수처럼 직관적으로 보이지 않는 수는 어떤 방식으로 정당화될 수 있는가?  
연산은 단순한 계산 절차인가, 아니면 수 사이의 관계를 드러내는 언어인가?  
계산 능력과 수학적 이해는 어떻게 구별될 수 있는가?

### 1-2. 대수와 방정식

방정식은 미지수를 찾는 도구인가, 아니면 관계를 압축해 표현하는 형식인가?  
같은 문제를 식으로 표현할 수 있다는 것은 어떤 종류의 추상화를 의미하는가?  
방정식의 해가 존재하지 않거나 여러 개일 때, 그것은 현실의 무엇을 말해주는가?  
대수적 조작은 기계적 절차인가, 아니면 구조를 보존하는 변환인가?  
문제를 방정식으로 바꾸는 순간 무엇이 보이고 무엇이 사라지는가?

### 1-3. 함수와 변화

함수는 입력과 출력의 대응인가, 아니면 변화의 구조를 표현하는 방식인가?  
현실의 복잡한 현상을 함수 하나로 나타낼 때 어떤 단순화가 발생하는가?  
선형 함수와 비선형 함수는 세계를 바라보는 관점을 어떻게 다르게 만드는가?  
함수의 그래프는 계산 결과를 보여주는가, 아니면 관계의 형태를 시각화하는가?  
함수 개념은 왜 현대 과학과 데이터 분석의 핵심 언어가 되었는가?

### 1-4. 기하와 공간

공간은 인간이 직관적으로 경험하는 것인가, 아니면 수학적으로 구성되는 대상인가?  
도형의 성질은 현실의 물체에서 비롯되는가, 아니면 이상화된 공간 안에서 정의되는가?  
유클리드 기하학은 왜 오랫동안 공간 이해의 표준으로 받아들여졌는가?  
비유클리드 기하학의 등장은 공간에 대한 인간의 확신을 어떻게 흔들었는가?  
그림으로 보이는 명백함과 논리적으로 증명된 사실은 어떻게 다른가?

### 1-5. 미분과 적분

미분은 순간의 변화를 포착하는가, 아니면 변화를 극한적으로 해석하는 방식인가?  
적분은 넓이를 구하는 기술인가, 아니면 무한히 작은 것들을 누적하는 사고인가?  
연속적인 변화는 실제로 존재하는가, 아니면 계산을 가능하게 하는 이상화인가?  
미분과 적분이 서로 역관계를 이룬다는 사실은 변화와 누적의 관계를 어떻게 보여주는가?  
미적분은 자연 현상을 설명하는 언어인가, 아니면 자연을 특정한 방식으로 재구성하는 틀인가?

### 1-6. 확률과 불확실성

확률은 무지의 정도를 나타내는가, 아니면 세계 자체의 불확실성을 표현하는가?  
동일한 사건의 확률을 사람마다 다르게 판단한다면, 확률은 객관적이라 할 수 있는가?  
낮은 확률의 사건이 실제로 발생했을 때, 우리는 그것을 우연이라 부르는가 필연이라 부르는가?  
확률적 사고는 인간의 직관을 교정하는가, 아니면 새로운 착각을 만들어내는가?  
불확실한 세계에서 합리적 선택이란 무엇을 기준으로 판단되어야 하는가?

### 1-7. 통계와 데이터

통계는 현실을 있는 그대로 보여주는가, 아니면 현실을 특정 기준으로 분류하고 요약하는가?  
평균, 중앙값, 최빈값은 같은 집단을 얼마나 다르게 보이게 만드는가?  
상관관계가 인과관계로 오해될 때 어떤 판단 오류가 발생하는가?  
데이터가 많아질수록 진실에 가까워지는가, 아니면 편향도 더 정교하게 증폭될 수 있는가?  
통계적 유의성은 실제 중요성과 어떻게 구별되어야 하는가?

### 1-8. 수열과 패턴

수열은 숫자의 나열인가, 아니면 반복과 변화의 규칙을 드러내는 구조인가?  
패턴을 발견하는 능력은 수학적 사고의 핵심인가, 아니면 과잉 해석의 위험을 동반하는가?  
등차수열과 등비수열은 현실의 어떤 성장과 감소를 모델링할 수 있는가?  
무한수열의 극한은 실제 도달점인가, 아니면 사고 속에서 정의되는 경계인가?  
인간은 왜 무작위 속에서도 패턴을 찾으려 하는가?

### 1-9. 논리와 증명

증명은 어떤 명제가 참임을 보이는 절차인가, 아니면 의심을 제거하는 사고의 형식인가?  
직관적으로 명백한 사실도 왜 증명이 필요한가?  
반례 하나가 보편 명제를 무너뜨릴 수 있다는 사실은 수학적 진리의 성격을 어떻게 보여주는가?  
귀납적 추론과 연역적 증명은 각각 어떤 한계와 강점을 가지는가?  
수학에서 참이라는 말은 현실과 일치한다는 뜻인가, 아니면 공리와 규칙 안에서 타당하다는 뜻인가?

### 1-10. 모델링과 응용수학

수학적 모델은 현실을 설명하는가, 아니면 현실을 계산 가능하도록 단순화하는가?  
모델이 유용하지만 정확하지 않을 때, 우리는 그것을 어떤 기준으로 평가해야 하는가?  
현실 문제를 수학 문제로 바꾸는 순간 어떤 변수들이 배제되는가?  
예측 모델의 실패는 수학의 한계인가, 아니면 모델링 과정에서 빠진 조건의 결과인가?  
응용수학은 순수한 사고의 확장인가, 아니면 현실을 통제하기 위한 기술인가?

## 2. 코딩

### 2-1. 프로그래밍의 본질

코딩은 컴퓨터에게 명령을 내리는 일인가, 아니면 문제를 절차로 번역하는 사고인가?  
좋은 코드는 실행만 잘 되는 코드인가, 아니면 사람이 읽고 수정할 수 있는 코드인가?  
프로그래밍 언어는 사고를 표현하는 도구인가, 아니면 사고의 방식을 제한하는 구조인가?  
같은 문제를 여러 언어로 풀 수 있다는 사실은 문제와 표현의 관계를 어떻게 보여주는가?  
코딩 능력은 문법 지식보다 문제를 분해하는 능력에 더 크게 좌우되는가?

### 2-2. 변수와 자료형

변수는 값을 담는 상자인가, 아니면 상태에 이름을 붙이는 방식인가?  
자료형은 컴퓨터의 제약인가, 아니면 의미 있는 조작을 가능하게 하는 규칙인가?  
동적 타입과 정적 타입은 개발자의 자유와 안정성을 어떻게 다르게 배열하는가?  
이름을 잘 짓는 일이 왜 코드의 이해 가능성에 결정적인 영향을 미치는가?  
데이터를 어떤 자료형으로 표현하느냐에 따라 가능한 사고가 달라지는가?

### 2-3. 조건문과 반복문

조건문은 선택을 구현하는가, 아니면 상황을 분류하는 규칙을 만드는가?  
반복문은 동일한 작업의 자동화인가, 아니면 패턴을 절차로 압축하는 방식인가?  
복잡한 조건이 많아질수록 프로그램은 왜 이해하기 어려워지는가?  
반복을 줄이는 코드는 언제 더 좋은 코드가 되고, 언제 오히려 과도한 추상화가 되는가?  
흐름 제어는 프로그램의 논리를 드러내는가, 아니면 숨겨진 예외를 만들어내는가?

### 2-4. 함수와 추상화

함수는 코드를 재사용하는 도구인가, 아니면 복잡성을 이름 아래 감추는 장치인가?  
좋은 함수는 짧은 함수인가, 아니면 하나의 책임을 명확히 가진 함수인가?  
추상화는 이해를 돕는가, 아니면 구체적 작동 방식을 은폐하는가?  
함수의 입력과 출력이 명확하다는 것은 왜 안정적인 설계의 출발점이 되는가?  
함수를 나누는 기준은 기계의 효율인가, 인간의 이해 가능성인가?

### 2-5. 자료구조

자료구조는 데이터를 저장하는 방식인가, 아니면 문제를 바라보는 관점인가?  
배열, 리스트, 스택, 큐, 해시맵, 트리는 각각 어떤 종류의 세계를 잘 표현하는가?  
좋은 자료구조 선택은 코드의 속도보다 사고의 명료성에 더 중요할 수 있는가?  
데이터의 삽입, 삭제, 탐색 방식은 프로그램 전체의 구조를 어떻게 결정하는가?  
현실의 관계를 자료구조로 바꾸는 순간 무엇이 단순화되고 무엇이 손실되는가?

### 2-6. 알고리즘과 복잡도

알고리즘은 문제 해결 절차인가, 아니면 효율성을 가진 사고의 형식인가?  
정답을 내는 알고리즘과 좋은 알고리즘은 어떻게 다른가?  
시간복잡도와 공간복잡도는 실제 성능을 얼마나 정확히 설명하는가?  
느린 알고리즘은 단순히 나쁜 것인가, 아니면 문제 규모에 따라 충분히 합리적일 수 있는가?  
효율성만을 기준으로 알고리즘을 평가할 때 어떤 가치가 배제되는가?

### 2-7. 객체지향과 설계

객체지향은 현실 세계를 코드로 모델링하는 방식인가, 아니면 변경을 관리하기 위한 설계 전략인가?  
클래스와 객체는 본질을 표현하는가, 아니면 역할과 책임을 임시로 묶는가?  
상속은 코드 재사용의 도구인가, 아니면 강한 결합을 만드는 위험한 구조인가?  
캡슐화는 정보를 숨기는 일인가, 아니면 잘못된 접근을 제한하는 설계인가?  
좋은 설계는 처음부터 완성되는가, 아니면 변경을 견디는 과정에서 형성되는가?

### 2-8. 디버깅과 오류

버그는 실수인가, 아니면 프로그램에 대한 잘못된 믿음이 드러나는 순간인가?  
오류 메시지는 실패의 알림인가, 아니면 시스템이 제공하는 진단 언어인가?  
디버깅은 코드를 고치는 일인가, 아니면 자신의 가정을 검증하는 과정인가?  
재현되지 않는 버그는 왜 가장 다루기 어려운 문제로 남는가?  
테스트는 버그가 없음을 증명하는가, 아니면 특정 조건에서 작동함을 확인하는가?

### 2-9. 웹과 데이터베이스

웹 애플리케이션은 화면을 만드는 일인가, 아니면 사용자 요청과 데이터 흐름을 설계하는 일인가?  
프론트엔드와 백엔드는 단순한 역할 분담인가, 아니면 서로 다른 책임 체계인가?  
데이터베이스는 정보를 저장하는 창고인가, 아니면 관계와 무결성을 관리하는 시스템인가?  
SQL과 NoSQL은 데이터의 구조를 바라보는 관점을 어떻게 다르게 만드는가?  
사용자 경험을 개선한다는 명목으로 수집되는 데이터는 어디까지 정당화될 수 있는가?

### 2-10. 보안과 윤리

보안은 공격을 막는 기술인가, 아니면 신뢰를 유지하기 위한 사회적 조건인가?  
개발자의 편의와 사용자의 안전이 충돌할 때 무엇을 우선해야 하는가?  
취약점은 코드의 결함인가, 아니면 시스템 전체 설계의 실패인가?  
개인정보를 처리하는 프로그램은 단순한 기능 구현을 넘어 어떤 윤리적 책임을 가지는가?  
자동화된 시스템이 피해를 만들었을 때 책임은 코드 작성자, 조직, 사용자 중 누구에게 있는가?

## 3. 경제

### 3-1. 희소성과 선택

경제학은 돈의 학문인가, 아니면 희소한 자원을 어떻게 배분할지 묻는 학문인가?  
희소성은 자연적 조건인가, 아니면 사회적 제도와 욕망이 함께 만들어낸 상태인가?  
선택에는 왜 항상 기회비용이 발생하는가?  
합리적 선택은 개인의 만족을 극대화하는 것인가, 아니면 제한된 조건 속에서 손실을 줄이는 것인가?  
무엇을 희소하다고 부르는 순간 어떤 가치가 우선순위에 놓이는가?

### 3-2. 시장과 가격

가격은 물건의 가치를 나타내는가, 아니면 수요와 공급이 만나는 순간의 신호인가?  
시장은 자원을 효율적으로 배분하는 장치인가, 아니면 힘의 차이를 은폐하는 교환 구조인가?  
가격이 오르면 항상 희소성이 커졌다고 말할 수 있는가?  
시장가격에 반영되지 않는 노동, 돌봄, 환경 비용은 어떻게 다루어져야 하는가?  
시장이 실패한다는 말은 시장이 없어진다는 뜻인가, 아니면 시장만으로 해결할 수 없는 문제가 있다는 뜻인가?

### 3-3. 수요와 공급

수요는 필요를 반영하는가, 아니면 구매력을 가진 욕망만을 반영하는가?  
공급은 생산자의 능력에 의해 결정되는가, 아니면 제도와 기대에 의해 조정되는가?  
수요와 공급의 균형은 안정된 상태인가, 아니면 끊임없이 이동하는 임시적 결과인가?  
수요가 있어도 구매력이 없다면 시장은 그것을 존재하지 않는 요구로 취급하는가?  
공급이 부족할 때 가격 인상은 효율적 신호인가, 아니면 취약한 사람을 배제하는 장치인가?

### 3-4. 화폐와 금융

화폐는 교환을 편리하게 하는 도구인가, 아니면 사회적 신뢰를 숫자로 표현한 제도인가?  
돈의 가치는 물질적 실체에서 나오는가, 아니면 집단적 믿음과 국가의 보증에서 나오는가?  
금융은 자본을 필요한 곳으로 이동시키는 장치인가, 아니면 미래의 기대를 현재의 가격으로 거래하는 시스템인가?  
부채는 단순한 빚인가, 아니면 미래의 시간을 현재에 끌어오는 방식인가?  
금융시장의 효율성은 실물경제의 안정성과 항상 일치하는가?

### 3-5. 성장과 분배

경제성장은 사회 전체의 풍요를 의미하는가, 아니면 특정 지표의 증가를 뜻하는가?  
성장이 계속되면 분배 문제는 자연스럽게 해결되는가?  
분배의 불평등은 개인 능력 차이의 결과인가, 아니면 제도적 배치의 효과인가?  
성장률이 높아도 다수의 삶이 나아지지 않는다면 그것을 성공이라 부를 수 있는가?  
경제정책은 파이를 키우는 일과 나누는 일을 분리해서 생각할 수 있는가?

### 3-6. 노동과 임금

임금은 노동의 가치를 정확히 반영하는가, 아니면 협상력과 제도의 결과인가?  
노동은 생계를 위한 수단인가, 아니면 인간이 사회적으로 인정받는 방식인가?  
생산성이 높아져도 임금이 오르지 않는다면 그 차이는 어디로 이동하는가?  
비임금 노동과 돌봄 노동은 왜 경제적 가치로 충분히 인정받지 못하는가?  
자동화가 노동을 대체할 때 인간의 소득과 존엄은 어떤 방식으로 재구성되어야 하는가?

### 3-7. 기업과 자본

기업은 상품과 서비스를 생산하는 조직인가, 아니면 위험과 책임을 분산시키는 제도인가?  
이윤 추구는 기업의 자연스러운 목적인가, 아니면 특정한 경제 체제가 부여한 규칙인가?  
주주 가치 극대화는 기업 운영의 합리적 기준인가, 아니면 다른 이해관계자를 주변화하는 원리인가?  
자본은 단순한 돈인가, 아니면 미래의 수익을 지배할 수 있는 권리인가?  
기업의 사회적 책임은 자발적 윤리인가, 아니면 법과 제도로 강제되어야 할 의무인가?

### 3-8. 국가와 재정정책

국가는 시장의 방해자인가, 아니면 시장이 작동할 조건을 만드는 행위자인가?  
세금은 개인 재산의 침해인가, 아니면 공동체 유지 비용의 분담인가?  
재정지출은 비효율적 낭비인가, 아니면 민간이 감당하지 못하는 위험을 흡수하는 장치인가?  
국가부채는 미래 세대에 대한 부담인가, 아니면 현재 필요한 투자를 가능하게 하는 도구인가?  
복지는 경제 활력을 약화시키는가, 아니면 불확실성을 줄여 사회 전체의 생산성을 높이는가?

### 3-9. 인플레이션과 경기변동

인플레이션은 단순한 물가 상승인가, 아니면 화폐 가치와 분배 구조의 변화인가?  
물가가 오를 때 모든 사람이 같은 고통을 겪는가?  
경기침체는 경제활동의 일시적 둔화인가, 아니면 기존 성장 방식의 한계를 드러내는 사건인가?  
중앙은행의 금리 결정은 물가만이 아니라 누구의 삶을 더 어렵게 만드는가?  
경기순환은 시장경제의 자연스러운 리듬인가, 아니면 정책과 심리가 함께 만드는 불안정성인가?

### 3-10. 국제경제와 불평등

무역은 모든 국가에 이익을 주는 교환인가, 아니면 산업 구조와 협상력에 따라 이익이 다르게 배분되는 체계인가?  
비교우위는 세계경제의 효율성을 설명하는가, 아니면 특정한 국제 분업을 정당화하는 논리로 쓰일 수 있는가?  
환율 변화는 수출입 가격만 바꾸는가, 아니면 한 국가의 노동과 자산 가치를 재조정하는가?  
세계화는 빈곤을 줄이는가, 아니면 새로운 형태의 의존과 불평등을 만드는가?  
국가 간 경제 격차는 자원의 차이 때문인가, 역사적 축적과 제도적 배치의 결과인가?
