**핵심 요약**

1. **휴리스틱(heuristic)**은 문제를 빠르게 풀기 위한 **경험적 규칙** 또는 **근사 방법**.
    
2. **최적해(optimal solution)**를 보장하지는 않지만 **계산을 크게 줄인다**.
    
3. 인간의 직관적 판단과 AI의 효율적 탐색에서 모두 사용된다.
    
4. 대표 예: **탐색 알고리즘(A*)**, **근사 최적화**, **의사결정 규칙**.
    
5. 핵심 목적: **복잡한 문제를 현실적인 시간 안에 해결**.
    

---

# 1. 휴리스틱의 기본 개념

문제 해결 방법은 크게 두 가지가 있다.

### (1) 정확한 알고리즘

모든 경우를 계산하여 **정답을 보장**한다.

예

- 모든 경우 탐색 (brute force)
    
- 동적 계획법
    

문제

- 계산량이 매우 커질 수 있음.
    

---

### (2) 휴리스틱 알고리즘

완벽한 계산 대신 **경험적 규칙**을 사용한다.

예

> “목표에 가까운 방향으로 먼저 탐색하자”

특징

- 빠름
    
- 계산 비용 낮음
    
- 최적해 보장 없음
    

---

# 2. AI에서 중요한 이유

많은 문제는 **완전 탐색이 불가능**하다.

예

- 체스
    
- 바둑
    
- 경로 탐색
    
- 일정 계획
    

가능한 경우의 수가 너무 많기 때문이다.

그래서 AI는

> **가능성이 높은 방향만 탐색**

하도록 휴리스틱을 사용한다.

---

# 3. 대표적인 휴리스틱 알고리즘

### 1️⃣ A* 탐색 알고리즘

경로 찾기 문제에서 많이 사용된다.

아이디어

- 지금까지 이동 비용
    
- 목표까지 예상 거리
    

를 합쳐서 탐색 우선순위를 정한다.

[  
f(n) = g(n) + h(n)  
]

- (g(n)) : 지금까지 실제 비용
    
- (h(n)) : 목표까지 예상 비용
    

---

### 2️⃣ 탐욕 알고리즘 (Greedy)

매 순간 **가장 좋아 보이는 선택**을 한다.

예

- 거스름돈 계산
    
- 최소 신장 트리
    

---

### 3️⃣ 유전 알고리즘

자연 진화에서 아이디어를 얻었다.

과정

1. 후보 해 생성
    
2. 좋은 해 선택
    
3. 교배
    
4. 돌연변이
    

반복하면서 점점 좋은 해를 찾는다.

---

# 4. 인간 사고도 휴리스틱 사용

인지 과학에서는 인간이 **휴리스틱으로 사고한다**고 본다.

대표 연구

- **카너먼 & 트버스키**
    

예

- 대표성 휴리스틱
    
- 가용성 휴리스틱
    
- 앵커링 효과
    

즉 인간은

> 모든 정보를 계산하지 않고 **빠른 규칙**으로 판단한다.

---

## 정리

휴리스틱 알고리즘은

> **“완벽한 계산 대신 빠른 추정으로 문제를 푸는 방법”**

이며

- 인간 인지
    
- AI 탐색
    
- 최적화 문제
    

에서 매우 중요한 개념이다.
